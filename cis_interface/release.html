Release Steps¶

  1. [on branch] Regenerate schema:

    $ cisschema
$ git add cis_interface/.cis_schema.yml
$ git commit -m "Updated .cis_schema.yml"
$ git push origin [BRANCH]

  2. [on branch] Open pull request to merge changes into master

  3. [on branch] Update docs to reflect new features and make sure they build locally:

    $ cd docs
$ make autodoc

  4. Merge branch into master after tests pass and code review complete

  5. Create tag with new version on master:

    $ git tag x.x.x

  6. Push to github (this triggers deployment of source distribution and wheels to PyPI):

    $ git push; git push --tags

  7. Update conda-forge build

  8. Checkout gh-pages branch if it dosn’t already exist (See below), regenerate docs and push to gh-pages branch:

    $ cd docs
$ make ghpages

Docs Checkout¶

Before running make ghpages, you must first checkout the ‘ghpages’ branch in the appropriate location. This only has to be done once on each machine you publish docs from. From the cis_interface source directory:

$ cd ../
$ git clone https://github.com/cropsinsilico/cis_interface.git cis_interface_docs
$ cd cis_interface_docs
$ git checkout gh-pages
